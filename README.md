# Мафия — Ведущий

**Онлайн:** [https://anyalink99.github.io/mafia-host-app.github.io/](https://anyalink99.github.io/mafia-host-app.github.io/)

Веб-приложение для ведущего спортивной «Мафии» на 10 игроков: раздача и показ ролей, стол с псевдонимами и статусами, таймер обсуждения, фолы и голосование с ничьими и «поднятием всех рук», **журнал партии** (голосования, казни, выбывания), экран **подведения итогов** с победителем, ролями, допами и правкой текстов лога, фоновая музыка по режимам, сохранение состояния в браузере. Можно установить как **PWA** или собрать **Android** через [Capacitor](https://capacitorjs.com) 8.

## Возможности

### Меню и сессия

- Главный экран: **раздать роли**, **к столу** без новой раздачи (если партия уже начата), **подведение итогов**, настройки, **сброс игры** (кнопка видна только при сохранённом состоянии).
- Состояние партии автоматически сохраняется в **`localStorage`** (`mafia_host_state`): роли, игроки и псевдонимы, очередь и сессия голосования, таймер, **журнал событий** (`gameLog`), итоги (победитель, роли в сводке, допы, имя ведущего и т.д.) — можно закрыть вкладку и продолжить позже.

**Кэш и обновления:** интерфейс и скрипты отдаются **сначала из сети** (сервис-воркер обновляет кэш в фоне), чтобы после деплоя не «залипала» старая вёрстка и логика. При смене версии сервис-воркера старый кэш удаляется при активации, страница **один раз перезагружается** под новый контроллер — офлайн по-прежнему работает за счёт актуального кэша. **При каждом коммите** увеличивайте `CACHE_NAME` в `service-worker.js` (см. комментарий в файле). `localStorage` хранит только данные партии и настройки музыки, а не разметку — если после обновления сайта всё ещё странное поведение, в DevTools → Application: **Unregister** service worker и **Clear site data**, затем обновите страницу.

### Раздача и карты

- Классический состав на **10 игроков**: 6 мирных, шериф, 2 мафии, дон — **случайная перетасовка** при новой раздаче.
- Карты с переворотом: на рубашке порядковый номер взятия, после открытия — «N игрок» в порядке показа; открытые карты можно **упорядочить по очереди выдачи**.
- Полноэкранный показ роли игроку с **SVG-иконками** (мирный / мафия / дон / шериф), закрытие по касанию (с короткой защитой от случайного тапа).

### Игровой стол

- Сетка игроков **№1–№10**: строка **псевдонима** (редактируется в карточке игрока), фолы (**+1**), при **4 фолах** — дисквалификация; отметка **выставлен** в очередь голосования.
- Действия по игроку (модальное окно): псевдоним, фол, выставить / **убрать с голосования**, снятие/переключение статусов **«Удалён»** (дисквалификация), **«Казнён»**, **«Убит»** — с иконками на ячейке; для выбывшего — **вернуть игрока** (с корректировкой журнала, где это применимо).
- **Таймер** обсуждения: пресеты **60 с** и **30 с**, старт / пауза; последние **10 секунд** таймер подсвечивается иначе.
- **Музыка** у таймера: режимы «Раздача карт» и «Ночные действия»; случайный трек из списка; если треков нет — подсказка загрузить файлы.

### Голосование

- **Очередь** кандидатов (минимум **два** для старта), отображение цепочки вида `3 → 7 → 1`, сброс очереди при исходе раунда.
- Ввод **числа голосов** по каждому выставленному через модальное окно с **сеткой быстрого выбора** и учётом **общего пула** живых игроков; после заполнения всех позиций — подсчёт.
- **Ничья** по максимуму: сужение до лидеров, **переголосование** (баннер на экране, таймер на 30 с), затем при повторной ничьей — этап **«голосование за поднятие всех рук»** (большинство от пула решает исход).
- Итоги голосований и казней попадают в **журнал партии** и отображаются на экране подведения итогов.

### Подведение итогов

- Поле **имени ведущего** (до 120 символов).
- Выбор **победителя**: не выбрано / мирные / мафия — после выбора разблокируются правки ролей и допов по игрокам (псевдоним можно менять всегда).
- **Сетка игроков**: номер, псевдоним, иконка роли; роль по умолчанию берётся из завершённой раздачи карт, при необходимости задаётся вручную (мирный, мафия, дон, шериф).
- По игроку в модалке: **дополнительные баллы** (ввод и шаги **±0,1**), **примечание к допу**, при необходимости — **«лучший ход»** (три номера игроков; поле ориентировано на первого убитого по журналу или уже заполненные тройки).
- Блок **«Ход игры»**: хронологический список событий из `gameLog` с автоматическим текстом; любую запись можно открыть и **переопределить текст** (сохраняется в событии как пользовательская подпись).

### Музыка в настройках

- Несколько пользовательских файлов на режим (**IndexedDB**), случайный выбор при старте; параметры трека: секунда старта, множитель громкости; встроенные демо-треки в папке **`audio/`** (метаданные в `localStorage`, ключ `mafia_host_music` — сброс партии их не удаляет).

## Требования

- [Node.js](https://nodejs.org/) (LTS) — для зависимостей Capacitor и скриптов `npm`
- Локальный HTTP-сервер для полноценной работы PWA, сервис-воркера и медиа (не открывать как `file://`)
- **Windows:** для `npm run icons` нужен PowerShell (скрипт ресайза иконок)

## Быстрый старт (веб)

```bash
git clone https://github.com/anyalink99/mafia-host-app.github.io.git
cd mafia-host-app.github.io
npm install
npx serve .
```

Откройте в браузере выданный адрес (`http://localhost:…`).

## Скрипты npm

| Скрипт | Назначение |
|--------|------------|
| `npm run copy:www` | Копирует статику в `www/` для Capacitor |
| `npm run cap:sync` | `copy:www` + `npx cap sync` |
| `npm run icons` | Генерация `icons/icon-192.png`, `icon-512.png` и `mipmap-*` для Android из `icons/icon.png` |

## PWA

После открытия по **HTTPS** или **localhost** браузер может предложить установку на главный экран.

1. Положите квадратный мастер-файл в `icons/icon.png` (желательно от 512×512).
2. На Windows: `npm run icons`.
3. При сборке под Android после смены иконок выполните `npm run cap:sync`, чтобы актуальные файлы попали в нативный проект.

Сервис-воркер **не** регистрируется в нативном приложении Capacitor (см. `js/main.js`).

**Версия кэша:** при каждом коммите увеличивайте `CACHE_NAME` в `service-worker.js` (например `mafia-host-static-v4` → `v5`), иначе после деплоя пользователи могут видеть старую вёрстку/скрипты из кэша.

## Android (Capacitor)

1. `npm install`
2. Если каталога `android/` нет: `npx cap add android`
3. `npm run copy:www`
4. `npm run cap:sync` (или отдельно `npx cap sync`)
5. `npx cap open android` — сборка APK/AAB в Android Studio

После правок HTML/CSS/JS или иконок перед сборкой снова выполняйте `npm run cap:sync`.

## Музыка

- У таймера на игровом столе кнопка с нотой: первый тап — выбор режима, повторный во время воспроизведения — стоп.
- В настройках для каждого режима можно добавить несколько файлов; при старте выбирается случайный трек.
- Параметры трека: секунда старта, множитель громкости 0.25–2 (итог ограничен `volume ≤ 1` у элемента `audio`).
- Ключ метаданных в `localStorage`: `mafia_host_music`. Сброс игры настройки музыки не удаляет.
